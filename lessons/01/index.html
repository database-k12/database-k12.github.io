  • a name
  • a data type.

The data must be stored in a lossless form. Designing the layout and storage of data based on the real-life scenario is called data modeling.

What if we use one table? !

!

Table: Knows-and-position

person position company knows
Steve Jobs CEO Apple Marissa Mayer
Steve Jobs CEO Apple Bill Gates
Bill Gates Chairman Microsoft Marissa Mayer
Bill Gates Chairman Microsoft Bill Joy
Marissa Mayer CEO Yahoo NULL
Bill Joy Chief Scientist SUN Microsystems NULL

!

Forcing everything to be stored as a single table causes some issues:

  1. Data redundancy: CEO and Apple are storied multiple times just because Steve Jobs knows lots of people.

  2. NULL values are used for Marissa Mayer and Bill Joy.

_______________________________

Let’s summarize the data model:

  • !

  • We will use two tables.

  • Table Knows will have two attributes:

    • person: text
    • knows: text

  • Table Position will have three attributes:

    • person: text
    • position: text
    • company: text
