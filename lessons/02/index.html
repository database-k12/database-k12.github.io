History

Invented in response to the increasing needs for data storage and data management from the growing financial banking industry. !

  • 1974: Invented by Edgar Frank Codd at IBM San Jose research lab.

  • 1977: Oracle Database

  • 1982: IngreSQL, UC Berkeley

  • 1983: IBM DB2

  • 1989: Microsoft SQL Server

  • 1995: MySQL

  • 1996: PostgreSQL

  • 2000: SQLite

  • 2009: Amazon Relational Data Service (RDS)

  • 2015: Google Big Table Service

Relational Database Management Systems (RDBMS)

!

From 1974 - now (over 40 years):

! The relational data model remains the most popular and time proven (1974 - now = over 40 years) data modeling framework.

!

! Structured Query Language (SQL) was and still is the industry standard in communicating with relational databases.

Elements of RDBMS

  • A database

    • name of the database
    • a collection of tables

    ! We will use a database that is already created as part of this Web page.

  • A table

    • name of the table
    • a list of attributes
    • a collection of tuples

Creating tables

CREATE TABLE <name> (
    <attribute> <type>, ...
);

  1. Specifies the attribute names and their types.

  2. Run multiple CREATE TABLE commands to create more than one table. !

Example:

CREATE TABLE cities (
    name text,
    population integer
);

Creating tuples

INSERT INTO TABLE <table>
VALUES (<tuple>), ... ;

Example:

CREATE TABLE IF NOT EXISTS cities (name TEXT, population INTEGER);

INSERT INTO cities VALUES ('Toronto', 2600000), ('Oshawa', 140000);

! Note the if not exists clause with create table.

SELECTING Tuples

What if we are only interested in some attributes of some tuples?

Use the select statement:

SELECT <attribute> ...
FROM <table>
WHERE <condition>

Example:

SELECT population FROM cities WHERE name = 'Toronto'

DELETING tuples

  • Tuples can be removed from a table using the DELETE command.

    DELETE FROM <table>
WHERE <condition>

  • Example

    DELETE FROM cities
WHERE name = 'Oshawa'
